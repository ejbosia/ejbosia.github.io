Unified Robotics II

The final project of Unified Robotics II consisted of navigating a maze and putting out a candle autonomously. My main responsibility for the project was to work on the mechanical systems. This included designing the drive system, determining sensors and sensor placement, and coming up with the method of moving the fan. I used SolidWorks to design the entire robot, with the custom pieces lasercut out of birch.

Lasercut Frame

One key aspect of the problem was the location of the flame must be displayed, measured from a starting location. To solve this problem, I decided to use a holonomic drive coupled with encoders to measure distance. This drive configuration is shown below. Because the maze was all right angles, the robot could navigate the right-angled maze without turning. We ran into some contact issues with the wheels because the maze surface was uneven, but we were able to partially correct the slipping by running a PID control loop on the comparison of the encoder ticks between each drive wheel. Because the robot only ever traveled straight, the ticks should have always been equal.

Holonomic Wheelbase

Aside from working on the mechanical components of the robot, I also created the high-level logic for the autonomous program. To control the robot, I used essentially a large switch statement with enumerated states. The figure below shows a snippet of C code used to control the robot autonomously. Each case has a list of function it calls in order to execute the program.
